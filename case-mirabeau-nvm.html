A Cloud-first strategy for the NVM

July, 2018 | Stephan van Stekelenburg | Case Study

The goal

Successfully migrate the complete NVM environment consisting of web sites, web services, back-end applications and associated databases to AWS. Cloud technology had to become the foundation of the digital services of the NVM towards its members.

The challenge

For the NVM to optimally operate from a cloud native platform, all its 75+ applications would have to be migrated to AWS in 1 day. To do this, the NVM’s platform has to be scalable so that cloud capacity grows with business, without infringing on the user experience of brokers.

The approach

We executed the largest IT migration for the NVM to date. After this lift & shift migration we optimized applications based on cloud-architecture concepts, deploying complementary cloud services and building or rebuilding portions using a cloud-native approach.

The result

Cloud technology from Amazon Web Services (AWS) forms the technological foundation. The applied cloud-first strategy offers a lot of value for the NVM in terms of flexibility, scalability, shorter development time and cost optimization. In addition, we are able to innovate faster with the use of cloud services, such as in the field of image recognition.

The details

Mirabeau is also responsible for the NVM’s entire digital platform’s management. 24/7 we monitor information flows within the platform and the behavior of the environment from an end-user perspective. Data integrity and security is paramount to the NVM –for which purpose, we can provide an ISAE 3402 type II statement. Mirabeau provides the NVM with various other services that facilitate the digital services of the NVM towards its members.

The NVM Media Solution

Real estate agents make extensive use of media for the sale of property. NVM's image library consists of 55 million media files and processes on average 60,000 items each day. To create a smooth process of sharing media with NVM members, we created a cloud native solution where different types of media can be processed and requested. The media platform was built quickly and cost-effectively by using ready-made building blocks.
